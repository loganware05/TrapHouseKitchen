2. **Create Blueprint:**
   - Click "New +" ã "Blueprint"
   - Select your repository: `traphouse-kitchen`
   - Render detects `render.yaml` automatically
   - Review resources created by the blueprint:
     - **`traphousekitchen-db`** ã PostgreSQL (top-level `databases` in `render.yaml`; `DATABASE_URL` on the API is wired with `fromDatabase`)
     - **`traphousekitchen-api`** ã Node backend
     - **`traphousekitchen-web`** ã static frontend
   - Click "Create Services"

3. **Configure environment variables (dashboard ã required):**

Render does not interpolate URLs in YAML. After services exist, set the **exact** public HTTPS URLs yourself (see [DEPLOYMENT_ENV_VARS.md](./DEPLOYMENT_ENV_VARS.md)).

**Backend (`traphousekitchen-api`):**
```bash
# CORS: must match the origin Mobile Safari sends (no trailing slash)
FRONTEND_URL=https://traphousekitchen-web.onrender.com

STRIPE_SECRET_KEY=sk_test_YOUR_KEY
STRIPE_PUBLISHABLE_KEY=pk_test_YOUR_KEY
STRIPE_WEBHOOK_SECRET=whsec_...
CHEF_EMAIL=your-email@example.com

# Optional (for emails):
RESEND_API_KEY=re_your_key_here
FROM_EMAIL=TrapHouse Kitchen <orders@yourdomain.com>
```

**Frontend (`traphousekitchen-web`) ã build-time (`VITE_*`):**
```bash
# Must be the public API base including /api; rebuild static site after any change
VITE_API_URL=https://traphousekitchen-api.onrender.com/api
VITE_STRIPE_PUBLISHABLE_KEY=pk_test_YOUR_KEY
```

After changing any `VITE_*` variable, **manually redeploy** the static site so the new values are baked into the bundle.

Use this after `FRONTEND_URL` and `VITE_API_URL` are set and the static site has been redeployed.

## Environment Variables

### Backend Variables

**From blueprint:**
- `DATABASE_URL` ã Injected via `fromDatabase` ã `connectionString` for `traphousekitchen-db` (see `render.yaml`). You do not paste this by hand for a fresh blueprint deploy.

**Set in dashboard (secrets / URLs):**
- `FRONTEND_URL` ã Public HTTPS origin of the static site, **no trailing slash** (used for CORS; must match what the browser sends, including Mobile Safari).
- `STRIPE_SECRET_KEY`, `STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET` ã Stripe Dashboard
- `CHEF_EMAIL` ã Email for order notifications

**Auto-generated / preset in blueprint:**
- `JWT_SECRET` ã Generated by Render
- `NODE_ENV` ã `production`
- `PORT` ã `10000`
- `JWT_EXPIRES_IN` ã `7d`

**Optional:**
- `RESEND_API_KEY` ã Email notifications
- Other keys as listed in [DEPLOYMENT_ENV_VARS.md](./DEPLOYMENT_ENV_VARS.md)

### Frontend Variables

All are **build-time** (`VITE_*`). Set in the static serviceãs Environment tab; **redeploy the static site** after changing them.

**Required:**
- `VITE_API_URL` ã `https://<your-api-host>/api` (public HTTPS URL, must end with `/api`)
- `VITE_STRIPE_PUBLISHABLE_KEY` ã Same mode as backend (`pk_test_...` vs `pk_live_...`)

### Pull request previews and CORS

`render.yaml` sets `pullRequestPreviewsEnabled: true` on the static site. Preview frontends get **different** `*.onrender.com` URLs than production. The API only allows the origin in `FRONTEND_URL`, so **API calls from a preview URL will fail CORS** unless you temporarily set `FRONTEND_URL` to that preview origin, turn off previews for the branch you test on device, or extend the backend to allow multiple origins (for example comma-separated `FRONTEND_URL` values ã implement in code if you choose that approach).

### How to Add Variables

1. Click service name (e.g., `traphousekitchen-api`)
2. Go to "Environment" tab
3. Click "Add Environment Variable"
4. Enter key and value
5. Click "Save Changes"
6. Service will automatically redeploy

## Database Setup

### Automatic migration

The API **build** runs Prisma migrations (`npx prisma migrate deploy` in `buildCommand` in `render.yaml`). That requires `DATABASE_URL` during build ã satisfied when the database is linked via `fromDatabase`. If you ever move migrate out of the build step, ensure the deploy environment still has DB access before the app starts.

### Seed Database (Optional)

To add initial data:

1. Go to `traphousekitchen-api` service
2. Click "Shell" tab
3. Run:
   ```bash
   cd backend
   npm run prisma:seed
   ```

**Alternative:** Use the seed SQL file from `backend/prisma/seed.sql`

### Add Custom Domain to Frontend

1. Go to `traphousekitchen-api` service
2. Click "Settings" ã "Custom Domain"
3. Add subdomain (e.g., `api.traphousekitchen.com`)
4. Update DNS records

## Troubleshooting

### Backend Deploy Fails

**Check:**
1. Go to `traphousekitchen-api` ã Logs
2. Look for error messages

**Common Fixes:**
```bash
# Missing environment variables
ã Add STRIPE_SECRET_KEY and other required vars

# Database connection failed
ã Check DATABASE_URL is set correctly

# Build errors
ã Verify TypeScript compiles locally: npm run build
```

### Frontend Shows API Errors

**Check:**
1. Is backend URL correct in `VITE_API_URL`?
2. Is CORS configured properly?
3. Check backend logs for errors

**Fix:**
```bash
VITE_API_URL=https://YOUR-BACKEND-URL.onrender.com/api
```

### Database Migration Fails

**Fix:**
1. Go to backend service ã Shell
2. Run manually:
   ```bash
   cd backend
   npx prisma migrate deploy
   npx prisma generate
   ```

### "Cannot find module '@prisma/client'"

**Fix:**
```bash
# In backend service shell:
cd backend
npx prisma generate
```

### Cold Starts (Free Tier)

On free tier, services "sleep" after 15 minutes:
- First request takes 30-60 seconds
